Smart Turret Template

Autonomous AI turret system for Unity. Drop-in detection, aiming, shooting, and destruction — all configurable from the inspector.

STT provides a self-contained turret that detects enemies by tag, rotates toward them via physics torque, fires a raycast-validated shot, and applies explosion-force damage to any STT_Actor target. No wiring required beyond assigning a muzzle transform and enemy tags.

DetectionSphere trigger with configurable radius and tag filtering
AimingPhysics torque rotation with damping and optional delay
ShootingRaycast line-of-sight validation, configurable fire rate
DamageToughness pool, armor threshold, explosion force knockback
EffectsShot FX, damage FX, destruction FX, audio clips
EditorCustom inspectors with runtime debug display for both components

Getting Started

Setting Up a Turret

  1. Drag the STT_Turret.prefab into your scene.
  2. In the References panel, assign the Muzzle transform (the barrel tip child object).
  3. In Parameters › Status, enable Active and Can Fire.
  4. In Parameters › Shooting, set Power, Delay, and Radius.
  5. In Targeting › Tags, add the tags of enemy GameObjects (e.g. Player, Enemy).
  6. In Effects, assign a Shot FX particle prefab and a Shot Clip audio clip.

Setting Up a Target

  1. Add STT_Actor to any GameObject that should be destructible.
  2. Ensure the GameObject has a Rigidbody and a Collider.
  3. Set the GameObject's tag to match one of the turret's Tags to Fire.
  4. Configure Toughness (health), Armor (collision threshold), and Damage Factor (knockback multiplier).
  5. Optionally assign Damage FX, Destroy FX, and a Destroy Clip.
Prefab The included STT_Turret.prefab ships with a pre-configured Rigidbody, SphereCollider (trigger), BoxCollider (body), AudioSource, and Animator. Open the demo scene STT_Overview.unity to see a working setup.

Turret Lifecycle

The full turret loop runs across two components. STT_Turret handles the AI side; STT_Actor handles the receiving end on any target (including the turret itself).

OnTriggerEnter → target added to list, aiming delay started
FixedUpdate → ClearTargets() → Aiming() → Invoke(Shooting)
Shooting() → Raycast → CheckTags → Shot() → ReceiveDamage()
ReceiveDamage() → toughness > 0: damage FX + knockback
                 → toughness ≤ 0: destroy FX + Die()
OnTriggerExit → target removed, next target selected
Target Switching When the current target is destroyed, ClearTargets() (called every FixedUpdate) removes it and automatically promotes the next target in the list. The turret never idles while enemies remain in range.

STT_Turret

The main turret MonoBehaviour. Requires Rigidbody, SphereCollider, BoxCollider, AudioSource, Animator, and STT_Actor on the same GameObject (all enforced via [RequireComponent]).

Parameters

Status

PropertyDefaultDescription
activefalseMaster switch. When false, all turret logic is skipped.
canFirefalseEnable shooting. The turret still aims when false.

Shooting

PropertyDefaultDescription
power—Damage dealt per shot. Also used as the explosion force magnitude applied to the target's Rigidbody.
ShootingDelay0.5–2 sPause between shots. Clamped to the [0.5, 2] range by the inspector slider.
radius—Sets both the detection trigger radius and the maximum raycast distance for shooting.

Targeting

Aiming

PropertyDefaultDescription
aimingSpeed—Torque multiplier applied to the Rigidbody per frame. Higher values rotate the turret faster toward its target.
aimingDelay0Seconds before the turret starts rotating after a target enters range. Set to 0 for instant tracking.

Tags

PropertyDescription
tagsToFireString array of Unity tags. Only colliders with a matching tag are added to the target list and shot at.
targetsLive list of all colliders currently inside the detection radius. Read-only at runtime.
targetThe currently active target (always targets[0]). The turret aims at and shoots this collider.
Tag Matching Tags are checked using CompareTag() which is allocation-free and avoids the performance cost of string comparison with ==.

Physics

PropertyDefaultDescription
rotationDamping5Angular damping applied to the Rigidbody on Awake. Prevents the turret from spinning past its target due to accumulated torque.
bodyColliderSize(2, 2, 2)Size of the BoxCollider used as the turret's physical body. Adjust to fit your mesh.
bodyColliderCenter(0, 1, 0)Center offset of the BoxCollider relative to the turret pivot.
Rigidbody Constraints The turret uses AddTorque for rotation. If the turret tips over, freeze the X and Z rotation axes on the Rigidbody component. The Y axis must remain free for horizontal aiming to work.

API Reference

void Aiming()

Called each FixedUpdate when isAiming is true and a target exists. Computes the angle between the turret's forward and the target, then applies torque via Rigidbody.AddTorque. Public so it can be called from external scripts to force an aim frame.

// The core aiming math
Vector3 delta = targeting.target.transform.position - transform.position;
float angle = Vector3.Angle(transform.forward, delta);
Vector3 cross = Vector3.Cross(transform.forward, delta);
rb.AddTorque(cross * angle * targeting.aimingSpeed);
void Shooting() private

Invoked after ShootingDelay seconds. Casts a ray from the muzzle; if it hits a collider with a matching tag, calls Shot() and ReceiveDamage() on the target's STT_Actor. Resets isShooting so the next shot can be queued.

void Shot() private

Plays the shot audio clip at a random volume (0.75–1.0), triggers the Shot animator trigger, and instantiates the shot FX prefab at the muzzle. The FX GameObject is destroyed after 2 seconds.

void ClearTargets() private

Iterates the target list backwards, removing any null or disabled colliders. Then sets targeting.target to targets[0] or null if the list is empty. Called every FixedUpdate.

STT_Actor

Add this component to any GameObject that should receive turret damage. Requires Rigidbody and AudioSource. Also responds to physical collisions — a fast-moving object crashing into an actor deals impact damage.

Parameters

PropertyDescription
toughnessThe actor's health pool. Each ReceiveDamage call subtracts from this. When it reaches zero, Die() is called.
armorMinimum collision velocity (m/s) required for a physics impact to deal damage. Acts as a threshold — light touches are ignored, hard hits are not.
damageFactorMultiplier applied to the explosion force on each hit. Higher values produce more knockback relative to the damage dealt.

Effects

VFX

PropertyDescription
damageFXParticle prefab instantiated at the hit position on each damage event. Destroyed after 3 seconds.
deactivateFXParticle prefab instantiated at the actor's position on death. Destroyed after 3 seconds.

SFX

PropertyDescription
destroyClipAudio clip played when the actor is destroyed. The clip plays in full before the GameObject is removed (2-second delay).

API Reference

void ReceiveDamage(float damage, Vector3 position)

Main damage entry point. Call from any system to deal damage. If isDead is already true, the call is ignored. If toughness - damage > 0, subtracts damage, applies explosion force at position, and spawns damage FX. Otherwise, calls Die().

// Call from any script that has a reference to the target
STT_Actor actor = hit.collider.GetComponent<STT_Actor>();
if (actor != null)
    actor.ReceiveDamage(50f, hit.point);
void Die()

Sets isDead = true, disables the collider and renderer, plays the destroy audio clip, and schedules Destroy(gameObject, 2). Safe to call multiple times — the guard flag prevents double-execution.

Naming This method is named Die(), not Destroy(). Using the name Destroy on a MonoBehaviour shadows UnityEngine.Object.Destroy, which can cause unexpected behaviour. Always call Die() to trigger destruction from external code.

Integration

Custom Targets

Any GameObject can be a turret target as long as it has:

  1. A Collider (any shape) with its tag set to match tagsToFire.
  2. A Rigidbody (required by STT_Actor for explosion force).
  3. An STT_Actor component with toughness > 0.

Objects without STT_Actor are safe to have in the scene — the turret's raycast only deals damage when GetComponent<STT_Actor>() succeeds on the hit collider.
